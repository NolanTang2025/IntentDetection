# 使用指南

## 🚀 快速开始

### 第一步：打开仪表板

1. **方式一（最简单）**：
   - 找到 `index.html` 文件
   - 双击用浏览器打开
   - 推荐使用 Chrome 或 Edge 浏览器

2. **方式二（推荐用于演示）**：
   ```bash
   # 在终端中进入文件夹
   cd visualization_dashboard
   
   # 启动本地服务器（Python 3）
   python3 -m http.server 8000
   
   # 在浏览器中访问
   # http://localhost:8000
   ```

### 第二步：浏览数据

打开后，您会看到：

1. **顶部统计卡片**：显示总用户数、片段数、聚类数
2. **导航标签**：5个功能模块
3. **数据可视化**：各种图表和详细分析

---

## 📊 功能模块详解

### 1. 总览（Overview）

**用途**：快速了解整体用户意图分布

**包含内容**：
- 用户聚类分布（饼图）
- 购买阶段分布（环形图）
- 价格偏好分布（柱状图）
- 核心关注点（水平柱状图）
- 关键洞察卡片

**使用场景**：
- 每日/每周数据回顾
- 向团队汇报整体情况
- 快速发现异常趋势

---

### 2. 用户聚类（Clusters）

**用途**：深入了解每个用户群体的详细特征

**使用步骤**：
1. 点击"用户聚类"标签
2. 在下拉菜单中选择要查看的聚类
3. 查看该聚类的详细信息

**包含内容**：
- 用户规模统计
- 浏览时长和交互次数
- 核心关注点
- 营销策略建议
- 产品推荐建议
- 转化优化建议
- 价格策略建议
- 内容策略建议

**使用场景**：
- 制定针对性的营销策略
- 了解特定用户群体的需求
- 优化产品推荐

---

### 3. 用户画像（Portraits）

**用途**：查看所有聚类的用户画像卡片

**包含内容**：
- 每个聚类的关键统计
- 核心兴趣
- 活跃时段

**使用场景**：
- 快速浏览所有用户群体
- 对比不同聚类的特征
- 发现用户行为模式

---

### 4. 业务洞察（Insights）

**用途**：查看每个聚类的完整业务建议

**包含内容**：
- 关键特征
- 营销策略建议
- 产品推荐建议
- 转化优化建议
- 价格策略建议
- 内容策略建议

**使用场景**：
- 制定营销计划
- 优化网站和页面
- 制定内容策略
- 制定价格策略

---

### 5. 产品偏好（Products）

**用途**：了解用户对不同产品的关注度

**包含内容**：
- 产品关注度排名（柱状图）
- 每个产品的详细统计
- 相关聚类信息

**使用场景**：
- 了解热门产品
- 优化产品推荐
- 制定产品营销策略
- 发现产品机会

---

## 💡 业务应用建议

### 日常运营

1. **每天早上**：
   - 查看"总览"页面，了解整体情况
   - 关注是否有新的趋势或异常

2. **每周分析**：
   - 深入查看"用户聚类"页面

   - 查看用户画像中的活跃时段

3. **产品推荐**：
   - 查看"产品偏好"页面
   - 根据聚类特征优化推荐算法

### 转化优化

1. **页面优化**：
   - 查看"业务洞察"中的转化优化建议
   - 针对不同聚类优化页面

2. **价格策略**：
   - 查看价格偏好分布
   - 根据价格策略建议制定促销活动

## 🔍 数据解读

### 如何理解聚类分布？

- **大聚类**（如聚类1有65个片段）：主要用户群体，需要重点关注
- **小聚类**（如聚类13只有2个片段）：特殊需求用户，可能需要个性化服务

### 如何理解购买阶段？

- **浏览阶段**：用户还在了解产品，需要教育性内容
- **对比阶段**：用户在选择产品，需要突出差异化优势
- **决策阶段**：用户准备购买，需要转化刺激

### 如何理解价格偏好？

- **高端**：对价格不敏感，更关注产品价值
- **中端**：需要性价比，对优惠敏感
- **经济型**：价格敏感，需要低价策略

## ⚠️ 注意事项

1. **数据更新**：
   - 数据文件 `data.js` 需要定期更新
   - 更新后刷新浏览器即可看到新数据

2. **浏览器兼容性**：
   - 推荐使用 Chrome 或 Edge
   - 确保浏览器支持 JavaScript

3. **数据隐私**：
   - 本仪表板包含用户数据
   - 请妥善保管，不要公开分享

## 🆘 常见问题

### Q: 页面显示空白或错误？
A: 检查 `data.js` 文件是否存在且格式正确

### Q: 图表不显示？
A: 检查网络连接，确保能访问 Chart.js CDN

### Q: 数据不准确？
A: 检查数据源文件是否正确更新

### Q: 如何导出数据？
A: 可以复制图表或查看 `data.js` 文件中的原始数据
